# Friseur-/Barbershop-Webseite - Übergabe-Checkliste

Diese Datei ist dafür gedacht, die neutrale Mustervorlage schnell an einen konkreten Salon anzupassen.

## Inhalte, die pro Salon eingesetzt werden

- Salonname und kurzer Claim.
- Logo als SVG, PNG oder hochauflösendes Bild.
- Echte Fotos: Außenansicht, Innenraum, Team, Arbeitsplatz, 3-6 Beispiel-Haarschnitte.
- Leistungen: finale Namen, Dauer, Preise und Kombi-Angebote.
- Zielgruppe und Stil: klassischer Friseur, moderner Barber, Premium-Salon, Unisex, Farbe, Pflege oder Grooming.
- Bewertungen: echte Google-Bewertungen mit Namen/Kürzel und Datum, nur mit erlaubter Nutzung.
- Kontakt: Telefonnummer, WhatsApp-Nummer, E-Mail-Adresse, Instagram/TikTok falls gewünscht.
- Adresse: Straße, Hausnummer, PLZ, Stadt und Google-Maps-Link.
- Öffnungszeiten: reguläre Zeiten, Feiertage, Urlaub, Sonderzeiten und Annahmeschluss.
- Rechtliches: Impressum, Datenschutz, Cookie-Hinweis falls Tracking oder externe Tools eingesetzt werden.

## Daten für die Terminbuchung

Damit Termine wirklich im Computerprogramm des Salons landen, werden diese Informationen gebraucht:

- Name des Terminprogramms, z. B. Treatwell, Shore, Planity, Salonized, Fresha, Phorest, Google Reserve oder eigenes Kassensystem.
- Gibt es einen öffentlichen Buchungslink?
- Gibt es ein einbettbares Widget oder nur eine externe Buchungsseite?
- Gibt es eine API oder Webhook-Funktion?
- Welche Services sind buchbar?
- Dauer pro Service, Preis pro Service und Pufferzeit zwischen Terminen.
- Welche Mitarbeiter dürfen für welche Services gebucht werden?
- Gibt es mehrere Stühle/Plätze parallel oder nur einen Kalender?
- Welche Kundendaten sind Pflicht: Name, Telefon, E-Mail, Notiz, Neukunde/Stammkunde.
- Wie werden Termine bestätigt: automatisch, manuell, per SMS, per E-Mail, per WhatsApp oder telefonisch?
- Stornoregel: bis wann darf ein Kunde absagen?
- Sollen No-Show-Regeln, Anzahlung oder Online-Zahlung eingebaut werden?
- Soll WhatsApp als Anfragekanal genutzt werden?

## Technische Livegang-Optionen

- Schnellste Variante: Button führt zum bestehenden Buchungslink des Salons.
- Gute Variante: Buchungswidget wird direkt auf der Seite eingebettet.
- Beste Variante: Formular sendet Daten über API/Webhook direkt ans Terminprogramm und zeigt freie Zeiten live an.

## Aktueller Stand der Mustervorlage

- Die Seite enthält bewusst keine echte Telefonnummer, Adresse oder echten Firmennamen.
- Das Terminformular ist eine Demo und sendet keine Kundendaten.
- Sobald Buchungslink, Widget oder API bekannt ist, kann der Terminbereich direkt angebunden werden.
